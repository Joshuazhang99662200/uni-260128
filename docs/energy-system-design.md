## 一、能量的定义和使用规则

### 1.1 什么是能量

**能量**是平台的核心虚拟货币，用于兑换平台提供的各类 AI Agent 智能服务。能量不可提现、不可转让，仅限本账户使用。

### 1.2 核心特性

| 属性 | 说明 |
|------|------|
| **有效期** | 永久有效，不清零 |
| **获取上限** | 单周最高 300 能量（周循环任务满额），单月最高约 1,200+ 能量 |
| **使用范围** | 仅限兑换平台 AI Agent 服务 |
| **可否转让** | 不可转让、不可提现 |

### 1.3 与其他权益的区别

| 类型 | 能量 | 折扣权益 |
|------|------|----------|
| 获取方式 | 完成各类任务 | 累计分享 5 次解锁 |
| 有效期 | 永久 | 终身 |
| 用途 | 兑换 Agent 服务 | 付费服务享 8.5 折 |

## 二、如何获取能量

### 2.1 新手/一次性任务（终身仅一次）

| 任务名称 | 能量奖励 | 说明 |
|----------|----------|------|
| 上传名片 | +50 能量 | 首次上传个人名片完成身份认证 |
| 完善基础信息 | +50 能量 | 补全个人资料（姓名、公司、职位等） |
| 累计分享 5 次 | 8.5 折权益 | 分享窗口无限制，解锁终身折扣（非能量奖励） |

### 2.2 周循环任务（跑环式·每周重置）

**核心机制：邀请好友（必须是新用户注册）参与上传 BP**

```
┌─────────────────────────────────────────────────────────────────┐
│                    周循环任务奖励阶梯                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   邀请人数        基础奖励        里程碑奖励       累计获得     │
│   ─────────────────────────────────────────────────────────────│
│   1 人            30              -               30 能量      │
│   2 人            60              -               60 能量      │
│   3 人 ⭐         90              +30             120 能量     │
│   4 人            120             -               150 能量     │
│   5 人 ⭐⭐       150             +120            300 能量     │
│                                                                 │
│   ⭐ = 里程碑节点                                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**重置规则：**
- 进度按**自然周**重置（周一 00:00 清零）
- 累计邀请人数、里程碑状态均重置
- 已发放的能量**不会**被收回

**月底促活机制：**
- 月底（29号前后）对未完成里程碑用户进行强提醒
- 示例消息：*"本周仅剩 2 天，您已邀请 3 人，再邀 2 人即可获得 120 额外能量！"*
- 利用"损失厌恶"心理刺激用户冲刺拉新

### 2.3 特殊任务

| 任务类型 | 能量奖励 | 触发条件 |
|----------|----------|----------|
| 问卷调查 | +20~50 能量 | 不定期发放，按问卷篇幅定价 |
| 反馈建议被采纳 | +20 能量 | 提交的产品建议被团队采纳实施 |

## 四、使用细则

### 4.1 能量发放规则

1. **即时到账**：完成任务后能量立即计入账户
2. **不可撤销**：已发放的能量不会因任务重置而收回
3. **记录可查**：所有能量收支记录均可在"我的能量"页面查看

### 4.2 能量消费规则

1. **先扣后用**：使用服务前先扣除相应能量
2. **余额不足**：能量不足时无法使用对应服务，需先获取能量
3. **不可退还**：服务开始后已扣能量不予退还
4. **无法透支**：账户能量余额不可为负

### 4.3 特殊说明

| 场景 | 处理方式 |
|------|----------|
| 邀请的好友未完成注册 | 不计入邀请人数 |
| 邀请的是老用户 | 不计入邀请人数（必须是新用户） |
| 周任务进度跨周 | 自然周重置，新周从 0 开始 |
| 服务使用异常 | 联系客服核实后可酌情退还能量 |
| 账号违规 | 平台有权冻结或清空能量账户 |

### 4.4 权益叠加说明

- **能量 + 8.5折权益可叠加使用**
- 示例：使用 100 能量兑换服务后，如需付费部分可享 8.5 折
- 折扣权益仅对**付费部分**生效，能量抵扣部分不参与折扣计算

### 4.5 反作弊机制

为保障平台公平性，以下行为将被视为违规：

- 使用虚假信息注册多账号刷能量
- 邀请非真实用户（机器人、小号等）
- 利用系统漏洞获取异常能量
- 其他扰乱平台正常运营的行为

**违规处理**：平台有权取消违规获得的能量、冻结账户或采取其他必要措施。

## 附录：能量系统数据结构建议

```typescript
// 能量记录类型
interface EnergyRecord {
  id: string;
  userId: string;
  amount: number;           // 正数为获取，负数为消耗
  type: EnergyType;
  taskId?: string;          // 关联任务ID
  serviceId?: string;       // 关联服务ID
  createdAt: Date;
  description: string;
}

// 能量类型枚举
enum EnergyType {
  // 获取类型
  UPLOAD_CARD = 'upload_card',           // 上传名片
  COMPLETE_PROFILE = 'complete_profile', // 完善信息
  INVITE_BASIC = 'invite_basic',         // 邀请基础奖励
  INVITE_MILESTONE_3 = 'invite_milestone_3',   // 3人里程碑
  INVITE_MILESTONE_5 = 'invite_milestone_5',   // 5人里程碑
  SURVEY = 'survey',                     // 问卷调查
  FEEDBACK_ADOPTED = 'feedback_adopted', // 反馈采纳

  // 消耗类型
  BP_DIAGNOSIS = 'bp_diagnosis',         // BP诊断
  MENTOR_PRACTICE = 'mentor_practice',   // 导师对练
  EQUITY_SIMULATION = 'equity_simulation', // 股权推演
  FINANCIAL_FORECAST = 'financial_forecast', // 财务预测
}

// 周任务进度
interface WeeklyTaskProgress {
  userId: string;
  weekStart: Date;          // 本周起始时间
  inviteCount: number;      // 本周邀请人数
  milestone3Claimed: boolean; // 3人里程碑是否已领
  milestone5Claimed: boolean; // 5人里程碑是否已领
}
```
